# 标题栏集成测试指南

## 已完成的修改

### 1. Tauri 配置 (`src-tauri/tauri.conf.json`)
- ✅ `decorations: true` - 保留原生窗口装饰
- ✅ `titleBarStyle: "Overlay"` - macOS 覆盖模式，允许内容延伸到标题栏区域
- ✅ `hiddenTitle: true` - 隐藏原生标题文字

### 2. QuickAccessToolbar 组件
- ✅ 添加 macOS 检测逻辑
- ✅ 添加 `data-tauri-drag-region` 属性（窗口拖动区域）
- ✅ 添加 `-webkit-app-region: drag` CSS（WebKit 拖动支持）
- ✅ 按钮设置 `-webkit-app-region: no-drag`（保持可点击）
- ✅ macOS 模式下左侧留出 80px 空间给红绿灯按钮
- ✅ 标题文字居中，macOS 下向右偏移 40px 以视觉居中
- ✅ 移除 "— LOGOS" 后缀，仅显示文档名

### 3. 样式调整
- ✅ 工具栏高度增加到 36px（容纳 macOS 红绿灯按钮）
- ✅ 全局 word-ribbon.css 同步更新

## 如何验证效果

### 在 Tauri 原生应用中查看（推荐）

当前已启动 `bun tauri dev`，应该能看到：

1. **macOS 上的效果：**
   - 窗口左上角有红绿灯按钮（●●●）
   - 快速访问工具栏（文件、保存、撤销等按钮）位于标题栏区域
   - 文档标题居中显示（例如："未命名文档"）
   - 可以通过拖动工具栏空白区域移动窗口
   - 点击按钮仍然正常工作

2. **Windows/Linux 上的效果：**
   - 标准窗口装饰
   - 工具栏位于内容区域顶部
   - 文档标题居中显示

## 故障排除

### 如果看不到效果：

1. **确认正在运行 Tauri 应用**
   ```bash
   # 应该看到原生窗口，而不是浏览器
   bun tauri dev
   ```

2. **检查 Tauri 版本**
   ```bash
   cd src-tauri
   cargo tree | grep tauri
   # 应该是 tauri v2.x
   ```

3. **清理并重新构建**
   ```bash
   cd src-tauri
   cargo clean
   cd ..
   bun tauri dev
   ```

4. **macOS 特定：检查系统偏好设置**
   - 确保"减少透明度"未启用
   - 系统偏好设置 > 辅助功能 > 显示
